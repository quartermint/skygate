<!-- OS captive portal check URL behavior (per D-07) -->
CNA (Captive Network Assistant) check flow:
  - iOS/macOS: GET http://captive.apple.com/hotspot-detect.html -> expects body containing "Success"
  - Android: GET http://connectivitycheck.gstatic.com/generate_204 -> expects 204 No Content
  - Windows: GET http://www.msftconnecttest.com/connecttest.txt -> expects body "Microsoft Connect Test"
  - When DNAT redirects these to Caddy, the Host header is preserved (DNAT rewrites IP, not Host)
  - If Caddy returns something other than expected response, OS opens CNA sheet showing the response
  - Strategy: match Host header for known CNA domains, serve captive.html -> OS opens CNA sheet with our portal

    **Step 4: Create pi/ansible/roles/dashboard/templates/Caddyfile.j2** (per RESEARCH.md Pattern 5, per D-07 captive portal OS check URL interception).

    CRITICAL: The Caddyfile MUST intercept OS captive portal check URLs (D-07). When nftables DNAT redirects HTTP from unauthenticated MACs to Caddy, the destination IP changes but the Host header is preserved. Caddy uses a `@captive_check` named matcher on Host header to detect CNA check requests from iOS (captive.apple.com), Android (connectivitycheck.gstatic.com), and Windows (www.msftconnecttest.com). These requests are served the captive.html page, which the OS CNA sheet renders, allowing the user to accept terms. Without this, the OS would receive index.html and may not trigger the CNA sheet properly.

    ```
    # Managed by Ansible - SkyGate Dashboard
    :{{ caddy_port }} {
        # OS captive portal check URL interception (D-07)
        # DNAT preserves Host header -- match known CNA check domains
        # iOS/macOS: captive.apple.com
        # Android:   connectivitycheck.gstatic.com
        # Windows:   www.msftconnecttest.com, msftconnecttest.com
        @captive_check {
            header Host captive.apple.com
            header Host connectivitycheck.gstatic.com
            header Host www.msftconnecttest.com
            header Host msftconnecttest.com
            header Host clients3.google.com
        }
        handle @captive_check {
            root * {{ dashboard_static_dir }}
            rewrite * /captive.html
            file_server
        }

        # API requests -> Go daemon
        handle /api/* {
            reverse_proxy localhost:{{ dashboard_port }}
        }

        # Captive portal page (direct access)
        handle /captive* {
            root * {{ dashboard_static_dir }}
            file_server
        }

        # Static dashboard files
        handle /static/* {
            root * {{ skygate_opt_dir }}
            file_server
        }

        # Default: serve dashboard index
        handle {
            root * {{ dashboard_static_dir }}
            try_files {path} /index.html
            file_server
        }
    }
    ```

    NOTE on Caddy `@captive_check` matcher: Caddy's `header` matcher within a named matcher block uses OR logic by default -- any matching header triggers the block. This is the correct behavior: we want to match ANY of the CNA check domains. Verify this during implementation by checking Caddy docs for named matcher semantics.

    ```
    # Managed by Ansible - SkyGate Dashboard nftables additions
    # Included from main nftables.conf

    # Devices that have accepted captive portal terms
    add set inet skygate allowed_macs { type ether_addr; flags timeout; timeout 24h; }

    # Per-device byte counters for dashboard
    add set inet skygate device_counters { typeof ether saddr; flags dynamic; counter; timeout 24h; }

    # Captive portal: redirect HTTP from unauthenticated devices
    add rule inet skygate prerouting iifname "{{ ap_interface }}" ether saddr != @allowed_macs tcp dport 80 dnat to {{ ap_gateway }}:{{ caddy_port }}

    # Track per-device bytes for authenticated devices
    add rule inet skygate forward iifname "{{ ap_interface }}" ether saddr @allowed_macs update @device_counters { ether saddr } accept
    ```
