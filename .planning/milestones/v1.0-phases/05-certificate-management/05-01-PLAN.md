<objective>
Extend the proxy server's certificate generation to support a root CA + intermediate CA delegation model, and harden the bypass domain list with hardcoded never-MITM categories.

Purpose: The root CA key must never leave the Pi (D-06, D-08). The remote proxy signs leaf certs using an intermediate CA signed by the root. Banking, auth, gov, health, and payment domains must always bypass MITM to avoid breaking cert-pinned apps (D-14). This plan builds the cryptographic foundation that Plans 02 and 03 depend on.

Output: Extended certgen.go with root CA (SSID-aware, 3-year, ECDSA P-256) and intermediate CA (1-year, MaxPathLen=0). Extended proxy.go with hardcoded bypass domains and BuildBypassSet merger. All tests passing.
</objective>

  <action>
    Add two new exported functions to certgen.go (keep existing LoadOrGenerateCA unchanged):

    1. `GenerateRootCA(certPath, keyPath, ssid string) (*x509.Certificate, crypto.PrivateKey, error)`:
       - Check if certPath and keyPath both exist. If so, load and parse them (return parsed x509.Certificate + key).
       - If both missing, generate new ECDSA P-256 keypair via `ecdsa.GenerateKey(elliptic.P256(), rand.Reader)`.
       - Create x509.Certificate template:
         - SerialNumber: random 128-bit from crypto/rand
         - Subject.Organization: []string{"SkyGate"}
         - Subject.CommonName: fmt.Sprintf("SkyGate-%s CA", ssid)   -- per D-07
         - NotBefore: time.Now()
         - NotAfter: time.Now().Add(3 * 365 * 24 * time.Hour)       -- 3 years per D-07
         - KeyUsage: x509.KeyUsageCertSign | x509.KeyUsageCRLSign
         - BasicConstraintsValid: true
         - IsCA: true
       - Self-sign with x509.CreateCertificate (template == parent since it's self-signed).
       - Write cert PEM to certPath (0644), key PEM to keyPath (0600).
       - Return parsed *x509.Certificate and crypto.PrivateKey.
       - If either file exists but the other doesn't (or files are corrupt), return error (same pattern as existing LoadOrGenerateCA).

    2. `GenerateIntermediateCA(rootCert *x509.Certificate, rootKey crypto.PrivateKey, certPath, keyPath string) (*tls.Certificate, error)`:
       - Check if certPath and keyPath both exist. If so, load via tls.LoadX509KeyPair and return.
       - Generate new ECDSA P-256 keypair.
       - Create x509.Certificate template:
         - SerialNumber: random 128-bit
         - Subject.Organization: []string{"SkyGate"}
         - Subject.CommonName: "SkyGate Intermediate CA"
         - NotBefore: time.Now()
         - NotAfter: time.Now().Add(365 * 24 * time.Hour)    -- 1 year
         - KeyUsage: x509.KeyUsageCertSign | x509.KeyUsageCRLSign
         - BasicConstraintsValid: true
         - IsCA: true
         - MaxPathLen: 0         -- can only sign leaf certs per Pitfall 4
         - MaxPathLenZero: true  -- explicitly zero
       - Sign with ROOT CA key (parent = rootCert, signerKey = rootKey) -- NOT self-signed.
       - PEM-encode cert and key, write to certPath (0644) and keyPath (0600).
       - Reload via tls.LoadX509KeyPair and return *tls.Certificate.

    Add necessary imports: "crypto", "crypto/x509" (if not already imported).

    - Test leaf cert signing with intermediate validates against full chain
