    **Step 2: Create pi/ansible/roles/dashboard/tasks/main.yml.**
    Tasks in order:
    1. Create static directory: `file: path={{ dashboard_static_dir }} state=directory mode=0755`
    2. Deploy dashboard daemon binary: `copy: src=../../bin/skygate-dashboard-linux-arm64 dest={{ skygate_opt_dir }}/skygate-dashboard mode=0755` -> notify restart skygate-dashboard
    3. Deploy static files: `synchronize: src=../../pi/static/ dest={{ dashboard_static_dir }}/ recursive=yes` OR individual copy tasks for each HTML/CSS/JS file -> notify reload caddy
    4. Deploy dashboard config: `template: src=dashboard.yaml.j2 dest={{ skygate_config_dir }}/dashboard.yaml mode=0644` -> notify restart skygate-dashboard
    5. Deploy domain categories: `copy: src=../../pi/config/domain-categories.yaml dest={{ skygate_config_dir }}/domain-categories.yaml mode=0644` -> notify restart skygate-dashboard
    6. Deploy Caddyfile: `template: src=Caddyfile.j2 dest=/etc/caddy/Caddyfile mode=0644` -> notify reload caddy
    7. Deploy systemd service: `template: src=skygate-dashboard.service.j2 dest=/etc/systemd/system/skygate-dashboard.service mode=0644` -> notify reload systemd, restart skygate-dashboard
    8. Deploy nftables additions: `template: src=nftables-dashboard.conf.j2 dest=/etc/nftables.d/dashboard.conf mode=0644` -> notify reload nftables
    9. Ensure Caddy is installed: `get_url: url=https://caddyserver.com/api/download?os=linux&arch=arm64 dest=/usr/local/bin/caddy mode=0755 creates=/usr/local/bin/caddy`
    10. Enable and start services: `systemd: name={{ item }} state=started enabled=yes` loop: [caddy, skygate-dashboard]

    ```
    # Managed by Ansible - SkyGate Dashboard
    :{{ caddy_port }} {
        # API requests -> Go daemon
        handle /api/* {
            reverse_proxy localhost:{{ dashboard_port }}
        }

        # Captive portal redirect page
        handle /captive* {
            root * {{ dashboard_static_dir }}
            file_server
        }

        # Static dashboard files

            root * {{ skygate_opt_dir }}
            file_server
        }

        # Default: serve dashboard index
        handle {
            root * {{ dashboard_static_dir }}
            try_files {path} /index.html
            file_server
        }
    }
    ```

    **Step 5: Create pi/ansible/roles/dashboard/templates/skygate-dashboard.service.j2.**
    Follow skygate-bypass.service pattern with these specifics:
    ```ini
    [Unit]
    Description=SkyGate Dashboard Daemon
    Documentation=https://github.com/quartermint/skygate
    After=network-online.target nftables.service pihole-FTL.service
    Wants=network-online.target pihole-FTL.service
    Requires=nftables.service

    [Service]
    Type=simple
    ExecStart={{ skygate_opt_dir }}/skygate-dashboard -config {{ skygate_config_dir }}/dashboard.yaml
    Restart=always
    RestartSec=5
    StandardOutput=journal
    StandardError=journal
    SyslogIdentifier=skygate-dashboard

    # Security hardening
    NoNewPrivileges=false
    ProtectSystem=strict
    ProtectHome=true
    ReadWritePaths={{ skygate_data_dir }}
    PrivateTmp=true

    [Install]
    WantedBy=multi-user.target
    ```

    **Step 6: Create pi/ansible/roles/dashboard/templates/nftables-dashboard.conf.j2.**
    nftables additions for captive portal (per RESEARCH.md Pattern 4):
    ```
    # Managed by Ansible - SkyGate Dashboard nftables additions
    # Included from main nftables.conf

    # Devices that have accepted captive portal terms
    add set inet skygate allowed_macs { type ether_addr; flags timeout; timeout 24h; }

    # Per-device byte counters for dashboard
    add set inet skygate device_counters { typeof ether saddr; flags dynamic; counter; timeout 24h; }

    # Captive portal: redirect HTTP from unauthenticated devices
    add rule inet skygate prerouting iifname "{{ ap_interface }}" ether saddr != @allowed_macs tcp dport 80 dnat to {{ ap_gateway }}:{{ caddy_port }}

    # Track per-device bytes for authenticated devices
    add rule inet skygate forward iifname "{{ ap_interface }}" ether saddr @allowed_macs update @device_counters { ether saddr } accept
    ```

    Same content as the template but with hardcoded paths (/opt/skygate, /data/skygate) for standalone use.
